Code to Design roundtrip

Import a React app into Stitch, edit the design visually, and export updated components — the full roundtrip.

The roundtrip workflow connects your codebase and Stitch in both directions. Start from a React app, extract its design system and screens into Stitch, make visual edits in the cloud, and export the updated designs back to clean React components.

What you'll build

Starting from an existing React + Vite application, you'll extract design tokens, capture screens as static HTML, upload everything to Stitch, edit the design, and convert it back to React components.

Prerequisites

Checklist

✓ Completed Get started with skills (skills installed, MCP connected)
✓ An existing React application with a working dev server
✓ Node.js 18+ and puppeteer installed

1 Extract design tokens from source

Scan your source files to reverse-engineer a design system document — no build step required.

PROMPT
Extract the design system from ./my-app/src and write it to .stitch/DESIGN.md

The extract-design-md skill reads your components, stylesheets, Tailwind config, and theme files. It identifies colors, typography, spacing, and component patterns.

What you'll see in .stitch/DESIGN.md

• YAML frontmatter with machine-readable color tokens
• Markdown sections describing visual atmosphere, palette, typography, and layout
• Descriptive names like "Deep Ocean Blue (#1a365d)" instead of raw hex codes

2 Capture static HTML snapshots

Start your dev server, then capture the running app as self-contained HTML:

PROMPT
Extract static HTML from http://localhost:5173 and save to .stitch/designs/home.html

The skill launches headless Chrome, captures the fully rendered DOM, inlines all CSS, and converts images to base64 data URIs. Each output file is completely self-contained — no external dependencies.

For multiple pages, repeat for each route:

PROMPT
Also extract http://localhost:5173/about to .stitch/designs/about.html

3 Upload to Stitch

Push everything to a new Stitch project in one command:

PROMPT
Upload .stitch/DESIGN.md and all HTML files in .stitch/designs/ to a new Stitch project called "My App Migration"

The skill creates a project, uploads the DESIGN.md as a design system, and imports each HTML file as a screen. Open Stitch to see your project with all the uploaded screens, styled according to your design system.

4 Edit the design

Make targeted visual changes without touching code:

PROMPT
Edit the home screen: add a testimonial section between the features and the footer with three customer quotes in a horizontal card layout.

The skill calls edit_screens with your instruction while preserving the existing design system. One change at a time produces the best results — iterate with additional edits rather than bundling everything into one prompt.

5 Export back to React

Convert the updated Stitch screens back to component code:

PROMPT
Convert the screens in my Stitch project to React components.

The react-components skill downloads the latest screen HTML, extracts Tailwind theme values, breaks the design into modular components, creates type-safe interfaces, moves static data into src/data/mockData.ts, and validates everything via AST.

Start your dev server — the React components now reflect the design changes you made in Stitch, including the new testimonial section.

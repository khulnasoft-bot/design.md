Create a Premium Design System

Generate curated, production-ready design systems that go beyond generic tokens — with semantic naming, complete coverage, and anti-generic standards.

The taste-design skill generates premium DESIGN.md files that enforce quality standards automatically. Instead of flat color names or utility values, it produces a rich, layered design vocabulary that makes AI-generated output consistent and on-brand.

What makes a design system premium

Semantic naming — Descriptive names like "Deep Ocean Blue" instead of "primary" or "blue-500". Every token name encodes context, hierarchy, and purpose.

Complete token coverage — All 8 Material 3 color roles are defined (primary, secondary, tertiary, error, surface, outline, fixed, inverse) plus an extended palette for backgrounds, overlays, and accent states.

Typography hierarchy — A 13-size scale with clear usage roles for each level: display, headline, title, body, and label, each with purpose-built font size, weight, line height, and letter spacing.

Component specs — Every component includes sizing, padding, border radius, background role, shadow elevation, and typography references — not just color values.

Visual rhythm — Consistent spacing scale (4px base unit) applied across margins, gaps, padding, and layout grids for visual harmony.

Using the taste-design skill

Give it a domain and get a complete premium design system back:

PROMPT
Create a premium DESIGN.md for a fintech dashboard

The skill analyzes the domain (fintech dashboard) and generates appropriate color palettes (trustworthy blues, greens for positive indicators), typography (data-dense table styles), and components (cards, charts, data tables) — all with premium naming conventions.

Anti-generic standards

The taste-design skill rejects generic output by default. Here's what "premium" means in practice:

Every token includes context — not just its value, but its role in the system. This means AI agents can make informed design decisions instead of guessing from flat values.

Quality enforcement

The taste-design skill enforces 0-warning output by default. If the generated design system has missing tokens, inconsistent naming, or undefined roles, it rejects and regenerates until all quality gates pass.

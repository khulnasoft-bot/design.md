Import from your codebase

Reverse-engineer a design system document directly from your existing source code — no build step needed.

The stitch::extract-design-md skill reads your components, stylesheets, Tailwind config, and theme files to create a semantic DESIGN.md. This lets you bring existing projects into Stitch's design workflow without starting from scratch.

Quick prompt (no skills needed)

If you already have the stitch-design plugin installed, just drop this into your agent:

PROMPT
Extract the design system from ./src and save it to .stitch/DESIGN.md

Using extract-design-md skill

The skill scans your codebase for design-related information across multiple sources:

Components — Parses React, Vue, or Svelte components for styling patterns, class name conventions, and recurring structures.

Stylesheets — Reads CSS, SCSS, and CSS Modules for custom properties, color definitions, typography rules, and media queries.

Tailwind config — Extracts the full theme configuration including custom colors, fonts, spacing scale, breakpoints, and animations.

Theme files — Detects design token files in various formats (JSON, JS, TS) and maps them to semantic names.

It then synthesizes everything into a single .stitch/DESIGN.md file with YAML frontmatter and human-readable markdown sections.

What gets extracted

CategoryWhat gets captured
ColorsHex values mapped to semantic names (e.g. #1a365d → surface-container)
TypographyFont families, sizes, weights, line heights, and letter spacing for each text style
SpacingPadding and margin scale values extracted from CSS and Tailwind config
ComponentsRecurring UI patterns with their styling properties and variants
EffectsGlows, gradients, shadows, backdrop filters, and other visual effects

Refining the output

The generated DESIGN.md is a starting point — you may want to adjust naming conventions, merge similar tokens, or add missing values.

Pro tip

Edit the resulting .stitch/DESIGN.md file directly to refine token names, adjust descriptions, or add new design tokens before using it with Stitch. The file is plain markdown with YAML frontmatter — easy to version control and review.
